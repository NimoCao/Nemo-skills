# Guided Writer Reference: Question Examples

## Purpose

This file gives `guided-writer` concrete questioning patterns.

These are not rigid scripts.
Use them as inspiration to ask sharper follow-up questions based on what the user has already said.

Use them like a good host would:
- ask the most useful next question, not every possible question
- stay sharp, but remain polite
- avoid turning objections into抬杠
- do not begin by offering candidate answers unless the user is stuck
- if the user gets stuck, offer candidate answers instead of repeating the same question endlessly
- expect the interview to close within about 6 to 8 questions

## General Follow-Up Moves

### 1. Topic Clarification

Use when the user starts broad.

Examples:
- "你最想写的不是这个领域本身，而是其中哪个判断？"
- "如果这篇文章最后只能留下一句话，你希望是哪一句？"
- "你现在更想写一个观点，还是一个经历？"

### 2. Trigger Excavation

Use when the user has a topic but no reason.

Examples:
- "为什么偏偏是现在想写这个？最近发生了什么？"
- "有没有一个具体瞬间，让你觉得这件事值得写下来？"
- "最开始让你不舒服、愤怒或者想明白的那个点是什么？"

### 3. Example Forcing

Use when the user keeps talking in abstractions.

Examples:
- "你能不能举一个具体例子，不要总结，直接讲那次发生了什么？"
- "如果把这个观点落到一个场景里，那一幕长什么样？"
- "有没有一个人、一句话、一次会议、一次失败，可以承接这个判断？"

### 4. Contradiction Digging

Use when the material lacks tension.

Examples:
- "这件事最反直觉的地方是什么？"
- "大家通常怎么理解它，而你为什么不同意？"
- "你亲身经历里，现实是怎么打脸原先判断的？"

If the user does not respond well to contradiction, switch to a softer version:
- "如果换个角度看，别人可能会怎么理解这件事？"
- "你觉得你这个判断最容易被误解成什么？"

### 5. Cost And Stakes

Use when the article还不够有重量.

Examples:
- "如果一直没想明白这个问题，代价是什么？"
- "你在这件事上真正付出过什么成本？"
- "如果别人继续用旧的理解，会在哪一步吃亏？"

### 6. Reader Positioning

Use when drafting方向开始模糊.

Examples:
- "你最想写给谁看？"
- "什么样的人会最需要这篇文章？"
- "读完之后，你希望读者改变的是想法、动作，还是情绪？"

## Question Ladders By Article Type

## A. 观点文

Goal:
Extract a sharp claim, opposing view, and support.

Suggested ladder:
1. "你真正想反驳的流行说法是什么？"
2. "你为什么不同意它？"
3. "你是在哪个具体场景里越来越确认这个判断的？"
4. "别人可能会怎么反驳你？"
5. "你准备怎么回应这种反驳？"
6. "如果把这篇文章浓缩成一个结论句，会是什么？"

## B. 个人经历文

Goal:
Get scene, feeling, turning point, and meaning.

Suggested ladder:
1. "这件事发生在什么时候？"
2. "当时你以为会怎样，后来实际怎样？"
3. "哪个瞬间最让你受影响？"
4. "你当时最真实的情绪是什么？"
5. "这件事后来改变了你什么？"
6. "你现在回头看，最想告诉当时的自己什么？"

## C. 经验方法文

Goal:
Extract problem, failed old method, improved method, and applicability.

Suggested ladder:
1. "你以前是怎么做的，问题出在哪？"
2. "你后来换成了什么做法？"
3. "这个变化最核心的不同点是什么？"
4. "有没有一次实践让你确认这个方法真的有效？"
5. "这个方法适合谁，不适合谁？"
6. "如果读者明天就试，第一步该怎么做？"

## D. 行业观察文

Goal:
Move from vague trend talk to real insight.

Suggested ladder:
1. "你看到的变化，到底是表面现象还是结构变化？"
2. "这个变化最容易被误读的地方是什么？"
3. "你是通过哪些案例意识到这个问题的？"
4. "哪些人会从这个变化里获益，哪些人会吃亏？"
5. "如果把你的观察讲成一句预测，那会是什么？"

## E. 情绪反思文

Goal:
Avoid empty抒情 and locate emotional truth.

Suggested ladder:
1. "这篇文章真正想处理的情绪是什么？"
2. "这个情绪第一次变得无法回避，是在哪个瞬间？"
3. "你表面上在说一件事，底层其实在害怕什么？"
4. "你有没有对自己撒过一个后来被拆穿的谎？"
5. "如果这篇文章不求解决，只求诚实，你最该写出的那句话是什么？"

## Sharpening Questions

Use occasionally to prevent soft, empty drafts.
These should sharpen the article, not create unnecessary confrontation.

Examples:
- "这句话听起来是对的，但还不够像你。你真正想说的狠话是什么？"
- "如果有人不同意你，他最可能反驳哪一点？"
- "你说这是经验，但你觉得它背后是个例，还是一种更普遍的现象？"
- "你现在讲的是结论。过程里最狼狈的部分是什么？"

## Recovery Questions

Use when the user gets stuck.

Examples:
- "没关系，我们先不追求完整。现在脑子里第一个画面是什么？"
- "如果不写成文章，只是跟朋友讲讲，你会怎么开头？"
- "先别总结，随便说一个你忘不掉的细节。"
- "如果这篇文章只能从一个例子开始，你会选哪个？"

## Offer Candidate Answers

Use when the same area has been asked once or twice and the user still cannot answer clearly.
Instead of继续追问, provide one to three candidate framings and let the user react.
Do not use this move too early, or it will narrow the user's thinking before their own judgment has surfaced.

Examples:
- "如果你一时说不清，我先替你收一下，看是不是这三个方向里的一个：A、B、还是 C？"
- "你刚才这个意思，我先帮你翻成一句更锋利的话，你看对不对。"
- "如果没有现成例子，我可以先给你编一个贴近你观点的例子，你来判断接不接近。"
- "这个点你不想展开也可以，那我先给一个可能的答案，你看看要不要认。"

## Signals To Start Drafting

You can usually draft when the conversation already contains:
- one clear claim
- one real example
- one tension or contradiction
- one intended audience or takeaway

If these are present and the user says "可以开始写了", stop asking and draft.
If you are already near the 8-question limit and these elements are mostly present, stop asking and draft.
