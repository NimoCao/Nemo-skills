# 全平台内容自适应

把一篇内容改造成可发布的平台包。默认输出：
- 小红书
- 微信公众号
- X / Twitter

最终交付默认做成 **HTML 发布工作台**，而不是只给 Markdown / 纯文本。HTML 页面里必须保留可直接复制发布的纯文本内容。

HTML 发布工作台要求：
- 生成一个独立 `.html` 文件，默认命名为 `platform-package.html`，放在该 topic 目录下。
- 页面按平台分区：小红书、微信公众号、X / Twitter、图片素材；如内容有可收藏方法论或提示词，单独增加“可收藏提示词 / 模板”分区。
- 每个主要文案块右上角都要有“复制”按钮，一键复制该块纯文本，不要把按钮文字一起复制进去。
- 图片素材要在页面内预览，并标注相对路径；图片仍需保存为 PNG，必要时同步 SVG。
- HTML 页面要适合本地直接打开，不依赖外网资源，不引入远程字体或 CDN。
- 视觉上要像一个发布工作台：清晰分区、可扫视、可复制、可预览，避免只有一大段长文。
- 如果用户明确要求 Markdown，再额外提供 `platform-package.md`；否则 HTML 交付就够了。为便于后续编辑，可以保留 Markdown 源文件作为内部备份，但最终给用户指向 HTML。

文案内容本身仍遵守：可直接复制发布，不使用 Markdown 标题语法、加粗、代码块、表格、引用块或图片嵌入语法。栏目名直接写“小红书标题”“公众号正文”等；在 HTML 中可以用视觉层级呈现这些栏目。

## 核心流程

1. 读懂原文
   - 提炼核心判断、读者痛点、反常识点、关键例子、金句。
   - 找出可做成图片的信息资产：长文字片段、清单、步骤、分类、对照、框架、检查项。
   - 保留作者真实判断，不要洗成泛泛的 AI 内容。

2. 拆成平台资产
   - 小红书：抓点击，降低阅读门槛，图片要能承载正文重点。
   - 微信公众号：保留完整观点链路，适合深读。
   - X：用自然英文提炼观点钩子，优先 thread。

3. 判断图片来源
   - 文字必须准确时，优先确定性文字渲染或设计图，不要硬用 AI 生图。
   - 真实界面、产品证据、聊天记录、数据反馈更有说服力时，建议用户自己截图/找图/拍摄。
   - 只有视觉故事、氛围、场景表达确实更适合生成图时，才调用生图。

4. 生成 HTML 发布工作台
   - 把平台内容拆成可复制卡片，而不是塞进单个长文本区。
   - 给正文、标题候选、提示词、Thread、发布建议等块加复制按钮。
   - 把已生成图片嵌入素材区，并保留路径说明。
   - 如果有“教程 / 模板 / 提示词 / 检查清单”类内容，要做成可收藏区，提高用户后续复用欲望。
   - 完成后在最终回复中优先给出 HTML 文件链接。

## 小红书图片优先级

默认给 1 张封面图 + 2-5 张内容图。

内容图优先级：
1. 备忘录图：最高优先级。适合大段文字、观点解释、文章摘录、个人判断、像苹果备忘录截图的小红书文字流。以后“小红书文字图”默认优先用这种。
2. 文字图 / 信息卡：低优先级。适合清单、对照、步骤、框架、表格、榜单等结构化干货。
3. 视觉图：适合故事场景、隐喻、前后变化，但仍要有辅助文字。

备忘录图要求：
- 3:4 竖版，推荐 1080x1440。
- 像苹果备忘录/系统笔记截图：浅色背景、白色或暖白纸张、圆角、顶部轻量系统栏、清晰标题、3-6 段正文。
- 文字是主角，单张可承载 300-700 字，但必须移动端可读。
- 适合用 `text-card-renderer` 的 `note` layout 直接导出 SVG + PNG。

小红书标题：
- 3-5 个，20 个中文字符以内。
- 覆盖情绪、反差、结果、身份代入、观点判断等不同角度。
- 标题和封面不要复读，要形成双钩子。

小红书正文：
- 默认 300-800 字。
- 开头直接给冲突、变化或判断，不铺长背景。
- 像真实创作者分享，短段落，适量 emoji。
- 结尾带互动问题和 5-8 个话题。

## 微信公众号

标题：
- 3-5 个，25 个中文字符以内。
- 克制可信，适合深读。

图片：
- 封面图：优先 900x383 或等比横版。
- 正文图：默认 2-4 张，至少 1 张解释型或收藏型信息图。
- 标注每张图的插入位置、尺寸、来源、文字、brief、生成路径或素材建议。

正文：
- 保留完整观点链路，不要压成摘要。
- 开头建立阅读理由。
- 小标题直接写纯文本行，不用 Markdown。

## X / Twitter

输出：
- X 内容方向
- 3-5 条英文单帖候选
- 3-7 条英文 thread
- 可选 0-2 张英文配图方案
- 发布建议

英文要像原生 X 写法，不要逐句硬译中文。每条默认控制在 280 characters 内。
